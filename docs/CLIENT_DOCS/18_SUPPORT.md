## 🚨 Emergency Support

### **When to Call Emergency Support**

**Call immediately if:**
- ✅ System completely down (cannot access)
- ✅ Cannot login (all users affected)
- ✅ Data corruption suspected
- ✅ Security breach detected
- ✅ Audit trail integrity failure
- ✅ Database down

**Do NOT call for:**
- ❌ Slow performance (non-critical)
- ❌ Single user login issue (use standard support)
- ❌ Question about features (use email)
- ❌ Billing question (use email)

## 📋 How to Report an Issue

### **Step 1: Gather Information**

Before contacting support, collect:

```
Essential Information:
├─ Your name and clinic
├─ Your user role (clinic, pharmacist, admin)
├─ When issue started (specific time/date)
├─ What were you trying to do?
├─ Exact error message (copy & paste)
├─ How often does it happen? (always, sometimes, once)
└─ Current status (still happening?)

Technical Details:
├─ Browser: Chrome 99 (or Firefox, Safari)
├─ Operating System: Windows 11, MacOS 12, etc
├─ Network: WiFi or mobile data?
├─ Device: Desktop, laptop, tablet, phone
├─ Affected users: Just you or everyone?
└─ Screenshot or video recording (if possible)

If Prescription-Related:
├─ Prescription ID (if known)
├─ Clinic name (originating)
├─ Clinic name
├─ Patient ID (last 4 digits only, for privacy)
└─ Timestamp of issue
```

### **Step 2: Try Basic Troubleshooting**

Before contacting support:

```
Try These First:
1. Refresh browser (F5 or Cmd+R)
2. Clear browser cache (Ctrl+Shift+Delete)
3. Try different browser (Chrome vs Firefox)
4. Try private/incognito mode
5. Logout and login again
6. Check internet connection (speedtest.net)
7. Restart your device
8. Wait 5 minutes and try again
```

---

### **Step 3: Submit Ticket or Call**

**For Email:**
```
Subject Line:
├─ Clear description
├─ Include user role
├─ Example: "Clinic: Cannot upload prescriptions"

Body:
1. What you were doing:
   "Trying to upload signed PDF"

2. What went wrong:
   "Got error: 'File rejected'"

3. How to reproduce:
   "Every time I try with any PDF"

4. Information collected:
   [Paste from Step 1]

5. Troubleshooting attempted:
   [Paste what you tried]
```

**For Phone:**
```
Have ready:
├─ Your clinic name
├─ Your name
├─ Your role
├─ Issue description (2-3 sentences)
├─ Error message (word for word)
└─ When it started
```

## 🔒 Security Issue Reporting

### **For Security Vulnerabilities**

**DO NOT:**
- ❌ Post on social media
- ❌ Tell other users
- ❌ Tell vendors
- ❌ Discuss in public channels

**DO:**
- ✅ Email: security@qesflow.com
- ✅ Use PGP encryption (key available on website)
- ✅ Include: Issue description, reproduction steps
- ✅ Provide: Proof of concept (minimal, don't exploit)

**Response Process:**
```
1. Receipt confirmed: Within 24 hours
2. Severity assessment: 1-3 business days
3. Fix development: Depends on severity
4. Testing: Before deployment
5. Deployment: To production (date provided)
6. Notification: You'll be informed of patch
7. Credit: Optional (your choice)
8. Embargo: Usually 90 days until public disclosure
```

**Severity Levels:**

| Severity | Example | Fix Time |
|----------|---------|----------|
| Critical | Full data breach | 24 hours |
| High | Authentication bypass | 48-72 hours |
| Medium | Privilege escalation | 1-2 weeks |
| Low | Information disclosure | 1 month |

## 📊 Support SLA (Service Level Agreement)

### **Response Time Targets**

| Priority | Severity | Response | Resolution |
|----------|----------|----------|-----------|
| P1 (Critical) | System down, data loss | 15 min | 4 hours |
| P2 (High) | Major feature broken | 1 hour | 8 hours |
| P3 (Medium) | Feature degraded | 4 hours | 24 hours |
| P4 (Low) | Minor issue, workaround | 24 hours | 5 days |

**Notes:**
- Response time: When we first contact you
- Resolution time: When issue is fixed/workaround provided
- Excludes weekends/holidays (unless P1)
- P1 is 24/7, others are business hours

## 💡 Tips for Getting Faster Support

```
DO:
✅ Be specific (not "it doesn't work")
✅ Provide exact error messages
✅ Include screenshots
✅ Describe what you were doing
✅ Be available to answer questions
✅ Try troubleshooting first
✅ Provide your location/timezone
✅ Be professional and respectful

DON'T:
❌ Be vague
❌ Use all caps
❌ Demand immediate resolution
❌ Threaten legal action
❌ Contact multiple channels simultaneously
❌ Provide patient data in emails
❌ Ask about other users' accounts
❌ Request back doors or special access
```
